專題計畫-倉儲機器人

本文最後更新於：2023年3月26日 上午

高三下的自主學習計畫

學習動機

  • 未來的趨勢

    • 在未來將會有許許多多的智慧工具產生，需多東西都逐漸走向自動化

  • 實際結合軟硬體

    • 在之前我們都只有接觸程式的部分，很少有同時接觸軟硬體的機會，藉由這次機會嘗試看看

  • 接觸新的領域

    • 我一直都在寫程式的小圈圈，藉由這次機會體驗看看3D列印、雷切等新的領域

  • 生活化

    • 在生活中智慧機器人扮演了很重要的角色，尤其是倉儲機器人為我們減少了許多不必要的繁瑣步驟藉由這次機會來了解他的實際運作模擬

學習方法

功能定位

  • 載物到定點

    • 能夠將物品安全的往返A點和B點

  • 自動化

    • 選擇完即可自動完成

  • 客製化位置

    • 能夠自訂要將物品載到哪個格子

程式撰寫

  • 使用arduino的官方API了解基本函式

    • 透過之前學過的C++基礎加以連接練習

    link:https://www.arduino.cc/reference/en/

  • 雷射切割

    • 這部份比較沒有甚麼經驗所以我們會先向老師詢問之後再靠自己去摸索完成
    • 使用軟體:inkscape

  • 3D列印

    • 這也是比較沒有經驗的部分，但好在身邊的朋友剛好友會的所以透過自己摸索和高人指點使得這項完成的輕鬆許多
    • 使用軟體:SketchUp

理想與現實

理想

  • 理想樣貌

    • 用循跡走道固定位置
    • 使用藍芽接收訊號
    • 要有升降平台(櫃子為3X3九宮格)
    • 能自己判斷並在接收到訊號後自動執行取物和放物

  • 理想狀況

    • 車車會重起點接收到目標的位置 -> 向前走到指定位置左轉 -> 升降至相對位置 -> 放入/取出 -> 後退到節點 -> 迴轉並回到起始位置 -> 升降平台初始化 -> 車車回到起始位置

現實

  • 現實樣貌

    • 配備
      • TT直流馬達*3
      • HC-06*1
      • L298N
      • Arduino UNO*1
      • 麵包版*1
      • 滑軌組*1
      • 5路循跡*1
      • 單路循跡*1
      • 免費的木頭和自製的3D列印配件
      • 3個年輕的肝

  • 現實狀況(並非為最終情況)

    • 紅色為未完成 / 綠色為完成
    • 車車會重起點接收到目標的位置 -> 向前走到指定位置左轉 -> 升降至相對位置 -> 放入/取出 -> 後退到節點 -> 迴轉並回到起始位置 -> 升降平台初始化 -> 車車回到起始位置

困難&解決

主機板連接埠不夠

  • 在做程式撰寫時發現一塊主板完全無法應付我們的需求

    solve: 我們將會用到的東西做大致整理，刪除一些多餘的線材，最後奇蹟般的剛剛好

疫情停課(前期)&製作時間太少(疫情後)

  • 停課使我們無法直接見面在製作過程中再成了很大的不便

    solve: (前期)約時間線上討論 / (疫情後)趁有空的時間約出來一起製作討論

3D列印結構問題

  • 印出來後因為結構問題直接斷裂

    solve: 加強結構做出V2、V3等版本

3D列印的擴充性

  • 做完時才發現有別的功能需求

    solve: 先完整討論完再重新設計一次到位

程式相容性

  • 有些程式會互相衝突

    solve: 換位思考修改內容並不斷嘗試

硬體設備故障或難用

  • 有些硬體會有些許的故障使整個車子呈現不穩定的情況

    solve: 這個比較大的原因是線材太爛，後來改了線材就好很多了。另一個問題是循跡當初也炸開了但後來使用了5路循跡之後除了使用到的線材量下降以外也使整個過成更加穩定
